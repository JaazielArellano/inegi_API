# inegi_API
Base para la API de consulta y actualización de datos normalizados de INEGI
# Tarea Tipo Examen
**Objetivo** : Migrar un conjunto de datos local (Excel normalizado INEGI) a una arquitectura en la nube, exponiendo la información a través de una API RESTful y consumiéndola en una herramienta de Business Intelligence para la toma de decisiones.
Herramientas a utilizar:
- **Base de Datos:** supabase (PostgreSQL).
- **Backend/API**: Python (Flask o FastAPI).
- **Hosting:** PythonAnyWhere.
- **Visualización (BI)**: tableau, Power BI o Looker Studio.
- **Validar API:** postman

#### Estructura de las Tablas (DB)
Diccionario de Código de Actividad:
```SQL
CREATE TABLE dic_codigo_actividad (
    id_cod_act INTEGER PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    descripcion_act VARCHAR(255) -- Aumentado a 255 por si las descripciones son largas
);
```
Diccionario tipo Vialidad:

```SQL
CREATE TABLE dic_tipo_vialidad (
    id_tip_vial INTEGER PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    desc_tip_vial VARCHAR(100)
);
```
Diccionario de Tipo de Asentamiento:
```SQL

CREATE TABLE dic_tipo_asentamiento (
    id_tip_asent INTEGER PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    desc_tip_asent VARCHAR(100)
);
```
**Tabla Ubicación**:
```SQL
CREATE TABLE ubicacion (
    id_ubicacion INTEGER PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    codigo_postal INTEGER,
    entidad VARCHAR(100),
    municipio VARCHAR(100),
    localidad VARCHAR(150),
    longitud DOUBLE PRECISION, -- Corregido de DOUBLE a DOUBLE PRECISION
    latitud DOUBLE PRECISION,  -- Corregido de DOUBLE a DOUBLE PRECISION
    id_tip_vial INTEGER,
    id_tip_asent INTEGER,
    CONSTRAINT fk_vialidad FOREIGN KEY (id_tip_vial) REFERENCES dic_tipo_vialidad(id_tip_vial),
    CONSTRAINT fk_asentamiento FOREIGN KEY (id_tip_asent) REFERENCES dic_tipo_asentamiento(id_tip_asent)
);
```
**Tabla Contacto**:

# Paso 2: Desarrollo de la API RESTful (Python)
## 
A travez de librerías como Flask accederemos a nuestra base de datos y haremos programaremos determinadas consultas a manera de KPI's, todo esto desde un entorno viertual contenido en tu maquina local, deberas seguir las siguientes instrucciones:

Primero: 
- Asegurate de tener python instalado.
- Creas una nueva carpeta y accedes a ésta desde el CMD de windows.
- Creas un entorno virtual con: `python -m venv nombre_entorno` y ahí mismo lo activas con `nombre_entorno\Scripts\activate`.
- Ahora te encargas de instalar las librerías necesarias `pip instal supabase` y además el programa **Flask**.
Segundo:
- Ahora nos encargamos de llenar el contenido de la carpeta, tiene que quedar con ésos 2 archivos:
![image.png](https://raw.githubusercontent.com/bucketio/img17/main/2026/05/03/1777858845062-f836958b-a700-496f-afed-997a9a9f633a.png 'image.png')
- El archivo ".env" escribiremos el link de nuestro proyecto Supabase y su llave de apli

```python
SUPABASE_URL=https://xyz.supabase.co
SUPABASE_KEY=tu_llave_aqui
```
![image.png](https://raw.githubusercontent.com/bucketio/img15/main/2026/05/03/1777858943703-f464eabe-4cb9-401d-8efb-876ed58ee169.png 'image.png')

- Después programaremos nuestro archivo main.py (que en nuestro ejemplo se llama "api_v1"), que contendrá las sig. consultas predefinidas:

```python
import os
from flask import Flask, request, jsonify
from supabase import create_client, Client
from dotenv import load_dotenv

load_dotenv()

app = Flask(__name__)

# Configuración de Supabase
url: str = os.environ.get("SUPABASE_URL")
key: str = os.environ.get("SUPABASE_KEY")
supabase: Client = create_client(url, key)

# 1. Catálogo General (GET /api/unidades)
@app.route('/api/unidades', methods=['GET'])
def get_unidades():
    response = supabase.table("negocio") \
        .select("id, nombre_establecimiento, razon_social") \
        .limit(50).execute()
    return jsonify(response.data)

# 2. Consulta por ID (GET /api/unidades/{id})
@app.route('/api/unidades/<int:id>', methods=['GET'])
def get_unidad_by_id(id):
    response = supabase.table("negocio").select("*").eq("id", id).single().execute()
    return jsonify(response.data)

# 3. Buscador por Nombre (GET /api/unidades/buscar)
@app.route('/api/unidades/buscar', methods=['GET'])
def buscar_por_nombre():
    texto = request.args.get('nombre', '')
    response = supabase.table("negocio") \
        .select("*") \
        .ilike("nombre_establecimiento", f"%{texto}%") \
        .execute()
    return jsonify(response.data)

# 4. KPI Estatal (GET /api/estadisticas/total_por_estado)
# Nota: Para esto consultamos la tabla de ubicación
@app.route('/api/estadisticas/total_por_estado', methods=['GET'])
def get_kpi_estatal():
    # Usamos una consulta RPC o consultamos y agrupamos en Python
    # En Supabase, lo más eficiente para contadores complejos es usar una View o RPC
    # Aquí una forma simple consultando la tabla ubicación:
    response = supabase.table("ubicacion").select("entidad").execute()
    import collections
    conteos = collections.Counter([d['entidad'] for d in response.data])
    return jsonify(conteos)

# 5. Filtros Múltiples Dinámicos (GET /api/unidades/filtro)
@app.route('/api/unidades/filtro', methods=['GET'])
def filtrar_unidades():
    estado = request.args.get('estado')
    actividad = request.args.get('actividad')
    
    query = supabase.table("negocio").select("*, ubicacion!inner(entidad)")
    
    if estado:
        query = query.eq("ubicacion.entidad", estado)
    if actividad:
        query = query.eq("id_cod_act", actividad)
        
    response = query.execute()
    return jsonify(response.data)

# 6. Perfil Completo Anidado (GET /api/unidades/{id}/perfil_completo)
@app.route('/api/unidades/<int:id>/perfil_completo', methods=['GET'])
def get_perfil_completo(id):
    # Gracias a la normalización, Supabase permite anidar con !inner o select simple
    response = supabase.table("negocio") \
        .select("*, ubicacion(*, dic_tipo_vialidad(desc_tip_vial)), contacto(*), dic_codigo_actividad(*)") \
        .eq("id", id).single().execute()
    return jsonify(response.data)

# 7. Búsqueda Geoespacial (GET /api/unidades/cercanas)
@app.route('/api/unidades/cercanas', methods=['GET'])
def get_cercanas():
    lat = float(request.args.get('lat'))
    lon = float(request.args.get('lon'))
    radio_km = float(request.args.get('radio', 5))
    
    # En una práctica de Cloud real, usaríamos PostGIS. 
    # Aquí simulamos un filtro de caja delimitadora (Bounding Box) simple
    delta = radio_km / 111.0 # Aproximación simple: 1 grado ~ 111km
    
    response = supabase.table("ubicacion") \
        .select("id_ubicacion, latitud, longitud") \
        .gte("latitud", lat - delta) \
        .lte("latitud", lat + delta) \
        .gte("longitud", lon - delta) \
        .lte("longitud", lon + delta) \
        .execute()
        
    return jsonify(response.data)

if __name__ == '__main__':
    app.run(debug=True)
```

Si quieres interactuar con los diferentes KPI's programados, debes agregar parámetrso al la ruta, tal y como lo muestra el sig. menú de rutas:

### **1. Catálogo General de Unidades**
- **Ruta**: GET /api/unidades
**Descripción**: Obtiene un listado simplificado de los negocios para una vista rápida.
**Cálculo/Consulta**: SELECT id, nombre_establecimiento, razon_social FROM negocio LIMIT 50.
**Retorno**: Un array de objetos con los 3 campos básicos.

### **2. Consulta de Detalle por ID**
- **Ruta**: GET /api/unidades/<id>
**Descripción**: Recupera toda la información almacenada en la tabla maestra negocio para un registro específico.
**Cálculo/Consulta**: Filtro directo por Primary Key (WHERE id = {id}).
**Retorno**: Un objeto con todos los campos de la tabla negocio.

### **3. Buscador por Nombre (Coincidencia Parcial)**
- **Ruta**: GET /api/unidades/buscar?nombre={texto}
**Descripción**: Busca establecimientos cuyo nombre contenga la cadena de texto proporcionada (sin importar mayúsculas/minúsculas).
**Cálculo/Consulta**: Operador ilike en SQL (WHERE nombre_establecimiento ILIKE '%{texto}%').
**Retorno**: Lista de negocios que coinciden con la búsqueda.

### **4. KPI: Unidades por Entidad Federativa**
- **Ruta**: GET /api/estadisticas/total_por_estado
**Descripción**: Resumen estadístico que contabiliza cuántos negocios hay en cada estado.
**Cálculo/Consulta**: Agrupación y conteo (SELECT entidad, COUNT(*) FROM ubicacion GROUP BY entidad).
**Retorno**: Un objeto donde las llaves son los nombres de los estados y los valores son el total de registros.

### **5. Filtro Dinámico Combinado**
- **Ruta**: GET /api/unidades/filtro?estado={X}&actividad={Y}
**Descripción**: Permite filtrar negocios por su ubicación geográfica (estado) y/o su tipo de actividad económica.
**Cálculo/Consulta**: JOIN entre negocio y ubicacion con condiciones opcionales AND según los parámetros enviados.
**Retorno**: Lista filtrada de unidades económicas.

### **6. Perfil Completo (Información Anidada)**
- **Ruta**: GET /api/unidades/<id>/perfil_completo
**Descripción**: La ruta más completa. Reconstruye el modelo relacional para mostrar el negocio con sus detalles de contacto y ubicación en un solo lugar.
**Cálculo/Consulta**: Múltiples JOINs (negocio + ubicacion + contacto + diccionarios).
**Retorno**: Un JSON jerárquico con sub-objetos ubicacion, contacto y actividad.

### **7. Búsqueda por Proximidad (Geolocalización)**
- **Ruta**: GET /api/unidades/cercanas?lat={X}&lon={Y}&radio={Z}
**Descripción**: Encuentra negocios dentro de un radio de acción circular (en kilómetros) a partir de un punto GPS.
**Cálculo/Consulta**: Cálculo de distancia euclidiana o Bounding Box sobre las columnas latitud y longitud de la tabla ubicacion.
**Retorno**: Lista de negocios con sus coordenadas que cumplen la condición de distancia.
